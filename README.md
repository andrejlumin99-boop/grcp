# 1. Оптимизация многопоточности и минимизация блокировок (SOLID / Производительность)
Для синхронизации потоков вместо тяжелого std::mutex используется std::shared_mutex (паттерн Single Writer / Multiple Readers):

GetParameter (Чтение): Использует std::shared_lock. Несколько потоков могут одновременно и без блокировок читать значения параметров, что критически важно для высоконагруженных систем.

SetParameter (Запись): Использует std::unique_lock. Монопольно блокирует хранилище только на момент обновления данных, гарантируя консистентность.

2. Защита от повторных запросов и утечек памяти (Безопасность)
Реализована строгая проверка уникальности request_id (идемпотентность метода SetParameter):

База обработанных ID организована в виде скользящего окна (Sliding Window Cache) фиксированного размера (MAX_HISTORY_SIZE).

При превышении лимита старые ID автоматически удаляются по принципу FIFO через std::deque. Это защищает сервер от деградации производительности и бесконечного потребления RAM (утечки памяти) при долгой работе под нагрузкой.

3. Современный C++ и RAII
Управление жизненным циклом ресурсов (мьютексы, потоки, сетевые каналы) полностью инкапсулировано в рамках концепции RAII (std::scoped_lock, std::unique_lock, std::shared_lock).

Передача тяжелых структур данных (например, при обновлении параметров) оптимизирована с помощью Move-семантики (std::move), что исключает лишнее копирование в памяти.

4. Работа со временем (Временные метки)
Временные метки (Timestamp) формируются на стороне сервера в момент получения данных с использованием стандартной библиотеки <chrono>. Время декомпозируется на секунды и наносекунды эпохи Unix, что обеспечивает независимость кода от системных кодировок ОС Windows и полную совместимость со спецификацией Protobuf.

5. Обработка ошибок
Система разделяет инфраструктурные ошибки gRPC (передаваемые через возвращаемый grpc::Status) и прикладной статус бизнес-логики (передаваемый в полях status_code и status_message ответа). Это позволяет клиентским приложениям гибко реагировать как на сетевые сбои, так и на логические ошибки (например, Parameter not found).
